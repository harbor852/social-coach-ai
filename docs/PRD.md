# AI 社交成长教练 PRD v0.1

## 1. 产品概述

### 1.1 产品名称
暂定名：**SpeakUp AI / 社交成长教练**

### 1.2 产品定位
面向低社交度人群的 AI 社交成长教练，帮助用户通过语音交流、角色扮演、表达训练、社交礼仪学习与复盘反馈，逐步提升社交自信、沟通能力、观点表达能力和真实场景中的互动表现。

产品不定位为心理治疗工具，也不替代心理咨询、医学诊断或危机干预。它更像是一个“低压力、可反复练习、不评判用户”的 AI 沟通陪练。

### 1.3 核心目标
帮助青春期至初入职场阶段的用户：

1. 克服不敢表达、不知道怎么开口、怕尴尬、怕被否定等社交障碍。
2. 学会在不同场景下自然、礼貌、有边界感地沟通。
3. 训练用户勇敢、有逻辑、有理有据地表达自己的观点。
4. 通过语音练习提升真实交流中的表达流畅度、自信感和临场反应。
5. 通过社交礼仪、情境模拟和复盘反馈，降低用户进入真实社交场景的心理门槛。

## 2. 背景与机会

### 2.1 用户痛点
目标用户常见问题包括：

- 不知道如何开启聊天。
- 和陌生人、同学、同事、异性、上级交流时紧张。
- 说话容易没有重点，表达混乱。
- 有观点但不敢说，怕说错、怕被嘲笑。
- 不懂得如何拒绝别人、提出请求、表达不满或争取权益。
- 社交礼仪不足，比如不会寒暄、不会接话、不会赞美、不会结束对话。
- 线上聊天还可以，线下语音或面对面表达明显紧张。
- 进入新学校、新公司、新团队时难以融入。
- 容易陷入讨好型表达，缺少边界感。
- 缺少安全、低压力、可反复练习的社交训练环境。

### 2.2 产品机会
传统社交训练通常依赖线下课程、心理咨询、社群活动或真实场景试错，门槛较高。AI Agent 可以提供：

- 随时可用的陪练对象。
- 不评判、低压力的训练环境。
- 多场景角色扮演。
- 个性化表达反馈。
- 长期成长记录。
- 语音交流训练。
- 针对性话术优化。

## 3. 目标用户

### 3.1 核心用户群体

#### A. 青春期用户
大致包括初高中、大学早期用户。

典型需求：

- 和同学相处。
- 课堂发言。
- 社团面试。
- 与老师沟通。
- 朋友关系维护。
- 面对被忽视、误解、冲突时如何表达。
- 提升自信和表达勇气。

注意：如果涉及未成年人，需要加入更严格的安全边界、内容限制和家长/监护人提示机制。

#### B. 大学生与准职场人
典型需求：

- 面试表达。
- 小组讨论。
- 课堂展示。
- 实习沟通。
- 与导师、同学、社团成员沟通。
- 公开表达观点。
- 建立人际连接。

#### C. 初入职场用户
典型需求：

- 向上汇报。
- 同事协作。
- 会议发言。
- 提出不同意见。
- 拒绝不合理请求。
- 争取资源。
- 处理冲突。
- 职场礼仪。

### 3.2 用户画像

#### 用户画像 1：不敢表达型
- 有想法，但开口前会反复担心。
- 害怕被否定或评价。
- 常常错过表达机会。
- 核心需求：练习勇敢表达、降低心理压力。

#### 用户画像 2：表达混乱型
- 想说很多，但没有结构。
- 表达时容易绕、啰嗦、没有重点。
- 核心需求：训练逻辑表达、观点论证和简洁表达。

#### 用户画像 3：社交尴尬型
- 不知道怎么开场、接话、结束话题。
- 容易冷场。
- 核心需求：破冰、接话、轻松互动、社交礼仪。

#### 用户画像 4：讨好退让型
- 不敢拒绝别人。
- 常常压抑自己的需求。
- 表达不满时容易过度道歉。
- 核心需求：边界感、坚定表达、拒绝训练。

#### 用户画像 5：职场新手型
- 不知道如何跟领导、同事、客户沟通。
- 会议上不敢表达观点。
- 核心需求：职场表达、汇报、会议发言、冲突沟通。

## 4. 产品价值主张

### 4.1 一句话价值主张
让低社交度用户在 AI 陪练中，逐步学会更自然、更自信、更有逻辑地表达自己。

### 4.2 核心价值

1. **低压力练习**：用户可以反复练，不怕丢脸。
2. **真实情境模拟**：覆盖校园、职场、朋友、家庭、陌生社交等场景。
3. **语音化训练**：不仅练文字，还练真实说话。
4. **观点表达训练**：帮助用户有逻辑、有依据地说出自己的想法。
5. **即时反馈**：指出表达中的问题，并给出优化版本。
6. **长期成长**：记录用户薄弱点与训练进步。
7. **安全边界**：不做心理诊断，不替代心理治疗。

## 5. 核心功能模块

## 5.1 社交场景分析模块

### 功能说明
用户输入一个社交困境，AI 帮助拆解场景、人物关系、用户目标、潜在风险和建议策略。

### 示例场景
- “我想和新同学聊天，但不知道怎么开口。”
- “领导在会上否定了我，我不知道怎么回应。”
- “朋友总是临时取消约定，我该怎么说？”
- “我喜欢一个人，但聊天总是很尴尬。”

### 输出结构
1. 当前场景判断。
2. 你的真实需求。
3. 对方可能的感受或立场。
4. 最合适的沟通目标。
5. 推荐表达策略。
6. 可直接使用的话术。
7. 可练习的小任务。

---

## 5.2 语音交流与实时陪练模块

### 功能说明
用户可以通过语音与 AI 对话，模拟真实交流。AI 可以扮演同学、朋友、上级、面试官、同事、客户等角色。

### MVP 形式
第一阶段采用“按住说话 / 点击录音”模式：

```text
录音 → 语音转文字 → Agent 分析 → 生成回复 → 语音合成播放
```

### 高级形式
第二阶段支持实时语音通话：

```text
WebRTC 实时语音 → Realtime Voice Layer → LangGraph Agent → 实时语音回复
```

### 子功能
- 语音输入。
- 语音回复。
- 自动转写字幕。
- 对话中断。
- 角色扮演。
- 语速反馈。
- 犹豫词统计。
- 表达清晰度评分。
- 自信感反馈。

### 反馈指标
- 语速是否过快或过慢。
- 停顿是否过多。
- 是否有大量“嗯、啊、就是、然后”。
- 观点是否清楚。
- 论据是否充分。
- 语气是否过度退让。
- 是否有礼貌和边界感。

## 5.4 观点表达训练模块

这是本产品的重要新增模块。

### 功能说明
训练用户勇敢、有逻辑、有理有据地表达观点，尤其适用于课堂发言、会议表达、面试、辩论、小组讨论、职场汇报等场景。

### 核心能力

#### A. 勇敢表达
帮助用户从“不敢说”过渡到“可以清楚表达自己的看法”。

训练内容：
- 如何开口表达不同意见。
- 如何在紧张时先说第一句话。
- 如何避免过度道歉。
- 如何表达不确定但有价值的观点。
- 如何在被打断后重新拿回话语权。

示例话术：

> “我有一个不同角度，可能不一定完整，但我想先提出来供大家参考。”

> “我理解你的观点，不过我这里有一个担心点，想补充一下。”

#### B. 逻辑表达
训练用户使用清晰结构表达观点。

推荐表达框架：

1. **PREP 结构**
   - Point：观点。
   - Reason：理由。
   - Example：例子。
   - Point：回到结论。

2. **结论先行结构**
   - 先说结论。
   - 再说理由。
   - 最后说建议。

3. **问题-原因-方案结构**
   - 问题是什么。
   - 为什么会发生。
   - 可以怎么解决。

4. **事实-感受-需求-请求结构**
   - 适合边界沟通、冲突沟通和亲密关系沟通。

#### C. 有理有据
帮助用户区分：

- 观点。
- 事实。
- 感受。
- 推测。
- 证据。
- 结论。

训练用户避免：

- “我就是觉得。”
- “大家都这么说。”
- “反正就是不行。”
- “你肯定是故意的。”

引导用户改成：

- “我的判断基于三个事实。”
- “目前我看到的数据是……”
- “我的担心是……”
- “所以我的建议是……”

### 典型训练场景
- 课堂上发表观点。
- 面试回答“你为什么适合这个岗位”。
- 会议中提出不同意见。
- 向领导争取资源。
- 和朋友表达不满。
- 与家人讨论自己的选择。
- 小组讨论时说服他人。

### 输出反馈维度
- 观点清晰度。
- 逻辑完整度。
- 证据充分度。
- 表达勇气。
- 语气稳定度。
- 说服力。
- 礼貌与边界感。

## 5.5 话术优化模块

### 功能说明
用户输入一句想说但不知道怎么说的话，AI 生成多个表达版本。

### 版本类型
- 温和版。
- 直接版。
- 高情商版。
- 自信版。
- 幽默版。
- 职场正式版。
- 边界清晰版。

### 示例
用户输入：

> “你老是不回我消息，我很烦。”

AI 输出：

温和版：

> “我发现我们最近回复节奏不太一样。你长时间不回的时候，我会有点不确定。我们可以说一下彼此舒服的沟通方式吗？”

边界版：

> “我能理解你忙，但如果经常没有回应，我会觉得这段沟通不太被重视。如果你最近不方便聊天，可以直接告诉我。”

高自尊版：

> “我更喜欢稳定、清楚的沟通。如果你暂时没有精力保持联系，也没关系，我们可以把节奏放慢。”

## 5.6 自信表达与魅力训练模块

### 功能说明
帮助用户减少自我否定、讨好型表达和过度解释，建立自然、松弛、有边界感的表达方式。

### 训练内容
- 如何自然介绍自己。
- 如何表达优点但不显得自夸。
- 如何接受赞美。
- 如何提出请求。
- 如何拒绝别人。
- 如何表达不同意见。
- 如何减少过度解释。
- 如何避免油腻、讨好或攻击性表达。

### 示例训练
用户原表达：

> “不好意思啊，如果你不方便就算了，我只是随便问问。”

优化表达：

> “我想邀请你一起参加这个活动。如果你有兴趣，我们可以约个时间。”

## 5.7 社交礼仪模块

### 功能说明
帮助用户学习不同场景下的基本礼仪、分寸感和互动规则。

### 模块内容

#### A. 基础社交礼仪
- 如何打招呼。
- 如何自我介绍。
- 如何寒暄。
- 如何接话。
- 如何结束对话。
- 如何表达感谢。
- 如何表达歉意。
- 如何赞美别人。

#### B. 校园社交礼仪
- 与老师沟通。
- 与同学合作。
- 宿舍关系处理。
- 社团面试礼仪。
- 小组作业沟通。

#### C. 职场礼仪
- 会议发言礼仪。
- 汇报礼仪。
- 邮件/即时消息礼仪。
- 向上沟通礼仪。
- 与同事协作礼仪。
- 拒绝不合理请求的礼仪。
- 饭局、团建、商务场景的基本分寸。

#### D. 线上沟通礼仪
- 微信/IM 消息怎么发。
- 多久追问一次合适。
- 如何避免冒犯。
- 如何表达清楚需求。
- 如何礼貌结束聊天。

#### E. 边界与尊重
- 不打探隐私。
- 不强行推进关系。
- 不用贬低式幽默。
- 尊重对方拒绝。
- 理解关系距离。
- 识别不舒服的互动。

### 训练方式
- 礼仪小课。
- 情境判断题。
- 错误示范与正确示范。
- 用户模拟练习。
- 场景复盘。

## 5.8 情绪识别与自我反思模块

### 功能说明
帮助用户识别自己在社交中的情绪与需求，但不做临床诊断。

### 可支持内容
- 识别紧张、害怕、委屈、愤怒、失落、尴尬等情绪。
- 引导用户区分事实、感受和猜测。
- 帮助用户发现自己的沟通目标。
- 帮助用户从“我是不是很差”转向“我可以怎么练”。

### 禁止内容
- 不诊断心理疾病。
- 不判断用户有人格障碍。
- 不替代心理咨询。
- 不给药物建议。
- 不做创伤治疗。

## 5.9 复盘反馈模块

### 功能说明
用户完成一次文字或语音练习后，系统给出结构化反馈。

### 反馈维度
- 清晰度。
- 逻辑性。
- 自信度。
- 礼貌度。
- 共情度。
- 边界感。
- 说服力。
- 表达自然度。
- 语音流畅度。

### 输出格式

```text
本轮表现评分：
- 清晰度：7/10
- 逻辑性：6/10
- 自信度：5/10
- 礼貌度：8/10

你做得好的地方：
1. 你表达了自己的真实想法。
2. 语气比较温和。

可以优化的地方：
1. 观点不够明确。
2. 缺少具体理由。
3. 结尾没有明确请求。

建议重说版本：
“我的建议是先采用 A 方案，因为它成本更低、上线更快，而且风险可控。我们可以先试运行一周，再根据反馈调整。”
```

## 6. 用户核心流程

  ↓

## 8. Agent 架构设计

### 8.1 推荐技术架构

```text
前端：React / Next.js / App
后端：FastAPI 或 Node.js
Agent 编排：LangGraph
工具层：LangChain
语音层：STT + TTS 或 Realtime API
数据库：PostgreSQL / Supabase
记忆系统：LangGraph checkpoint + 用户画像表
知识库：RAG + 向量数据库
评估观测：LangSmith / 自建日志系统
```

### 8.2 LangGraph 节点设计

```text
用户输入
  ↓
Safety Node 安全识别
  ↓
Intent Node 意图识别
  ↓
Scene Node 场景识别
  ↓
Emotion Reflection Node 情绪与需求识别
  ↓
Strategy Node 沟通策略生成
  ↓
Expression Coach Node 表达训练
  ↓
Roleplay Node 角色扮演
  ↓
Feedback Node 复盘反馈
  ↓
Memory Node 记忆更新
```

### 8.3 Agent 类型

#### 主教练 Agent
负责整体对话、训练引导和结果整合。

#### 场景分析 Agent
负责理解人物关系、场景背景和沟通目标。

#### 表达教练 Agent
负责优化话术、训练逻辑表达和观点表达。

#### 礼仪教练 Agent
负责不同场景下的社交规则与礼貌边界。

#### 角色扮演 Agent
负责扮演对方角色，制造真实互动。

#### 安全守护 Agent
负责识别心理危机、自伤、伤害他人、未成年人风险和不当内容。

## 10. 安全与伦理边界

### 10.1 产品边界
产品提供沟通训练、表达反馈、社交礼仪和自我反思，不提供临床心理诊断或治疗。

### 10.2 风险场景
以下场景需要触发安全流程：

- 用户表达自伤、自杀、严重绝望。
- 用户表达伤害他人的意图。
- 用户遭遇霸凌、虐待、性骚扰、严重胁迫。
- 未成年人涉及高风险关系或不安全线下见面。
- 用户表现出严重心理危机。

### 10.3 安全响应原则
- 停止普通社交训练。
- 表达支持和重视。
- 鼓励联系可信任的人。
- 建议联系当地紧急服务或专业人士。
- 对未成年人场景，建议联系监护人、老师或可信成年人。

### 10.4 内容限制
产品不能帮助用户：

- 操控、欺骗、PUA 他人。
- 规避对方拒绝。
- 制造情感依赖。
- 进行骚扰式追求。
- 伪装身份欺骗他人。
- 使用攻击、羞辱或威胁话术。

1. 不做完整实时语音通话。

4. 不做临床心理评估。
5. 不做深度人格测试。
6. 不保存原始语音，除非用户主动授权。

## 15. 差异化设计

### 15.1 与普通聊天机器人的区别
普通聊天机器人主要给建议，本产品强调“练习—反馈—重练—成长记录”。

### 15.2 与心理咨询产品的区别
本产品不做治疗，而是做沟通训练、表达训练和社交礼仪学习。

### 15.3 与课程类产品的区别
课程是单向学习，本产品是互动陪练，能根据用户表现即时调整。

### 15.4 与陪聊产品的区别
陪聊偏情绪陪伴，本产品偏能力训练和行为改善。

## 16. 风险与应对

### 风险 1：用户把产品当成心理医生
应对：明确产品边界，加入安全提示和危机转介机制。

### 风险 2：用户学习操控他人的话术
应对：加入伦理守护，不提供操控、欺骗、骚扰、PUA 类型建议。

### 风险 3：未成年人使用风险
应对：设置年龄识别、内容限制、监护人提示、安全场景分流。

### 风险 4：AI 反馈伤害用户自尊
应对：反馈采用支持性语言，强调可训练行为，不评价人格。

### 风险 5：语音隐私风险
应对：默认不保存原始音频；如需保存，必须单独授权。

## 17. 推荐 MVP 用户体验示例

### 场景：用户想练习会议发言

用户：

> “我开会时不敢表达不同意见。”

AI：

> “我们先练一个低压力版本。我会扮演你的同事，提出一个方案。你的目标不是反驳我，而是用礼貌、有依据的方式补充一个不同角度。”

AI 扮演同事：

> “我觉得这个活动就按上次方案来就行，没必要再改。”

用户语音回答：

> “嗯……我觉得可能还是要改一下吧，因为感觉上次效果也一般。”

AI 反馈：

```text
你已经表达了不同意见，这是好的开始。
但这句话有三个问题：
1. 观点不够明确。
2. 理由太模糊。
3. 缺少替代建议。

你可以这样说：
“我建议这次不要完全沿用上次方案。原因是上次报名转化率只有 18%，低于预期。如果我们这次把宣传重点改成用户案例，可能会更容易吸引目标人群。”

我们再练一遍，我继续扮演同事。
```

## 18. 总结

这个产品方向可行，并且具备明确的用户痛点和可训练场景。最关键的产品设计不是让 AI “给建议”，而是让 AI 成为一个可以反复练习、持续反馈、逐步建立自信的社交成长教练。

建议优先聚焦三个核心能力：

1. **敢说**：帮助用户克服开口障碍。
2. **会说**：帮助用户掌握礼貌、清晰、有边界的表达。
3. **说得有力量**：帮助用户有逻辑、有理有据地表达观点。

第一版 MVP 可以围绕“社交场景分析 + 观点表达训练 + 语音角色扮演 + 复盘反馈”构建。社交礼仪、自信表达、成长计划可以作为增强模块逐步加入。
